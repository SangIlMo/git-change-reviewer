# Data Model: Claude Code PR Annotation Skill Plugin

**Feature**: 002-claude-code-skill
**Date**: 2026-04-02

These are conceptual data structures that the SKILL.md prompt and helper scripts work with. There is no persistent storage — all data lives in memory within a single analysis session.

---

## Entities

### PRContext

Collected by `fetch-context.sh`. Represents the full metadata for a pull request.

```json
{
  "repo": "owner/repo",
  "pr_number": 123,
  "title": "feat: add user authentication",
  "description": "Implements JWT-based auth flow...",
  "commit_messages": [
    "feat: add login endpoint",
    "feat: add JWT token generation",
    "fix: handle expired token edge case"
  ],
  "base_branch": "main",
  "head_branch": "feat/user-auth",
  "head_sha": "abc123def456"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `repo` | string | GitHub repository in `owner/repo` format |
| `pr_number` | integer | Pull request number |
| `title` | string | PR title |
| `description` | string | PR body/description text |
| `commit_messages` | string[] | All commit message headlines in the PR |
| `base_branch` | string | Target branch (merge destination) |
| `head_branch` | string | Source branch (PR branch) |
| `head_sha` | string | HEAD commit SHA of the PR branch |

---

### DiffFile

A single file within the PR diff. Produced by parsing the unified diff output of `gh pr diff`.

```json
{
  "path": "src/auth/login.ts",
  "status": "modified",
  "hunks": ["<ChangeBlock>", "..."]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `path` | string | File path relative to repo root |
| `status` | enum | `added` / `modified` / `deleted` / `renamed` |
| `hunks` | ChangeBlock[] | Ordered list of change blocks in this file |

**Skip conditions**: binary files (detected by `Binary files ... differ` in diff), lock files (`*.lock`, `package-lock.json`, `yarn.lock`, `Cargo.lock`), generated files (`*.min.js`, `*.min.css`, `dist/*`, `*.pb.go`, `*.generated.*`).

---

### ChangeBlock

A semantically coherent unit of change within a single hunk. Produced by `parse-hunks.sh`. Each block is independently annotated.

```json
{
  "file_path": "src/auth/login.ts",
  "start_line": 42,
  "end_line": 58,
  "content": "+  const token = jwt.sign(payload, secret, { expiresIn: '24h' });\n+  return token;",
  "block_type": "addition"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `file_path` | string | File path (same as parent DiffFile) |
| `start_line` | integer | First line number in the new file |
| `end_line` | integer | Last line number in the new file |
| `content` | string | Raw diff lines (`+`/`-`/` ` prefixed) for the block |
| `block_type` | enum | `addition` / `deletion` / `modification` |

---

### IntentAnnotation

Generated by Claude Code (the AI engine) for each ChangeBlock. Represents the analyzed intent before being converted to a ReviewComment.

```json
{
  "file_path": "src/auth/login.ts",
  "start_line": 42,
  "end_line": 58,
  "body": "Adds JWT token generation with 24-hour expiry. The `expiresIn` value aligns with the session timeout requirement specified in the PR description.",
  "source_type": "context",
  "confidence": "high"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `file_path` | string | File this annotation applies to |
| `start_line` | integer | First line of annotated range |
| `end_line` | integer | Last line of annotated range |
| `body` | string | Human-readable intent explanation |
| `source_type` | enum | `context` (from commit/PR text) / `inferred` (AI-derived) / `conflict` (commit vs diff mismatch) |
| `confidence` | enum | `high` / `medium` / `low` |

---

### ReviewComment

The final payload structure for the GitHub PR Reviews API. Derived from IntentAnnotation by `post-review.sh`. Matches the GitHub API schema.

```json
{
  "path": "src/auth/login.ts",
  "line": 58,
  "start_line": 42,
  "side": "RIGHT",
  "start_side": "RIGHT",
  "body": "> **Intent**: Adds JWT token generation with 24-hour expiry.\n> **Source**: PR description\n> **Confidence**: High"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `path` | string | File path (required by GitHub API) |
| `line` | integer | Last line of the comment range (required) |
| `start_line` | integer | First line of the comment range (omit for single-line) |
| `side` | string | Always `"RIGHT"` (new file side) |
| `start_side` | string | Always `"RIGHT"` when `start_line` is set |
| `body` | string | Formatted annotation text (Markdown) |

---

### PluginManifest

The content of `.claude-plugin/plugin.json`. Describes the plugin to the Claude Code marketplace.

```json
{
  "name": "annotate-pr-plugin",
  "description": "Analyzes PR changes and posts AI-generated intent annotations as review comments",
  "version": "1.0.0",
  "author": {
    "name": "SangIlMo"
  },
  "repository": "https://github.com/SangIlMo/git-change-reviewer",
  "license": "MIT",
  "keywords": ["pr", "review", "annotation", "code-review", "ai"],
  "skills": ["./skills/annotate-pr/SKILL.md"]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Unique plugin identifier (kebab-case) |
| `description` | string | One-line description shown in marketplace |
| `version` | string | Semver version string |
| `author` | object | Author info with `name` (and optional `email`) |
| `repository` | string | GitHub URL for the plugin source |
| `license` | string | SPDX license identifier |
| `keywords` | string[] | Marketplace search tags |
| `skills` | string[] | Relative paths to SKILL.md files in this plugin |

## Data Flow

```
gh pr diff + gh pr view
        │
        ▼
    PRContext + raw diff
        │
   fetch-context.sh
        │
        ▼
   unified diff text
        │
   parse-hunks.sh
        │
        ▼
  ChangeBlock[] (JSON)
        │
   SKILL.md (Claude)
        │
        ▼
  IntentAnnotation[] (JSON)
        │
   post-review.sh
        │
        ▼
  ReviewComment[] → gh api POST /reviews
```
